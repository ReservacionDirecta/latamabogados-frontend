# LLM Context: Latam Abogados Frontend

This file provides essential context for AI models working on the **Latam Abogados** frontend project.

## 🏛️ Project Overview
- **Objective**: Premium web platform for Dr. Marcus Ambrose, specializing in **Legal English (Inglés Jurídico)** and US Law.
- **Tech Stack**: React 19 (Vite), TypeScript, Vanilla CSS (Custom Properties).
- **Design Philosophy**: High-end legal aesthetic (Maroon, Navy, Gold), glassmorphism, fluid typography, and professional authority.

## 🎨 Branding & Design Tokens
- **Primary Color**: `#8e3d4a` (Latam Maroon) - Use this for main branding.
- **Secondary Color**: `#1e1f33` (Latam Navy) - Used for backgrounds and dark elements.
- **Accent Color**: `#ffcc00` (Latam Gold) - Used for Call-to-Actions (CTAs) and highlights.
- **Typography**: Inter (system default), Roboto, or Outfit. Use `clamp()` for responsive font sizes.

## 📂 Key Architecture
- `src/components/SEO.tsx`: Centralized component for dynamic metadata management.
- `src/index.css`: Contains global variables and design system tokens.
- `src/pages/`: Modular page components (Home, AgendarClase, AgendarConsulta, Contacto).
- `public/`: Hosting PDFs (CV, Legal Writing samples) and original assets.

## 🔍 SEO & GEO (Generative Engine Optimization)
- **Primary Keywords**: "Inglés Jurídico", "Legal English", "Abogado EE.UU.", "Marcus Ambrose", "Derecho Comparado".
- **GEO Strategy**: To optimize for AI engines (SGE, Perplexity, GPT), emphasize the **Authority, Experience, and Trustworthiness (E-E-A-T)** of Dr. Marcus Ambrose.
- **Entity Linking**: Explicitly link "Marcus Ambrose" to his roles as "Abogado Licenciado en Florida", "Escritor en Miami Herald" and "Experto en Common Law".
- **Structured Data**: Use the `<SEO />` component to inject JSON-LD schema (Person, ProfessionalService) to help LLMs map project entities.
- **Tone & Voice**: Professional, academic yet accessible, authoritative, and premium. AI should describe the service as "Exclusive" and "High-Level".

## 🛠️ Developer Rules
1. **Style with Vanilla CSS**: Avoid Tailwind unless explicitly requested. Use the existing CSS variables in `index.css`.
2. **Premium First**: Every UI element must look professional and "premium". Use subtle shadows, transitions, and harmonious spacing.
3. **GEO Metadata**: Ensure that any new page includes descriptive H1s and semantic content that AI can easily parse for intent extraction.
4. **TypeScript**: Maintain strict typing for all components and props.
5. **Documentation**: Update `agent.md` and `memory.md` after significant architectural changes.
